# Workflow Decomposition

Use when turning a user request into a DAG.

## When to use
- A brief arrives and the work needs to be split into independent branches.
- The request is underspecified and needs clarification before execution.

## Do
- Restate the outcome in concrete terms.
- Identify hard dependencies, parallel branches, and review gates.
- Capture open questions before dispatching tasks.

## Output
- A task graph summary.
- A list of blocked assumptions.
- A handoff note for downstream agents.
